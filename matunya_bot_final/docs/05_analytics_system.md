ольше оставаться на `SQLite`.
- **Достаточность для Аналитики:** Мы по-прежнему можем строить всю основную статистику (процент правильных ответов по темам и навыкам), анализируя `AnswerLog` по `task_id`.
- **Фундамент для Будущего:** Наличие полных данных по ошибкам позволяет в будущем строить более сложные системы анализа (например, AI-тьютора, который ищет **типы** ошибок).

## 3. Техническая Реализация

- **Модели:** Описаны в `utils/models.py`.
- **Логика работы с БД:** Инкапсулирована в `utils/db_manager.py`. Хендлеры вызывают только его функции (`log_answer`, `get_student_stats`).
- **Изменение структуры БД:** Только через миграции `Alembic`.
- **Первичное наполнение `tasks`:** Через скрипт `scripts/populate_tasks_db.py`.
